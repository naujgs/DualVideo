## Spacing Scale

Native SwiftUI uses pt (points), not px. All values map directly to Swift `CGFloat` constants.

**Grid note:** This is a native iOS/SwiftUI project on a 4pt grid. The standard token set (4, 8, 16, 24, 32, 48, 64) is extended with 12, 20, and 28 as explicit project tokens — all are multiples of 4 and are grid-aligned. These extensions are required by the existing codebase layout contract and matching the iPhone Camera app proportions.

| Token | Value (pt) | Usage in this phase |
|-------|-----------|---------------------|
| xs | 4 | Zoom preset button vertical padding |
| sm | 8 | Zoom preset button horizontal inner gap (`HStack(spacing: 8)`) |
| md | 12 | RecordingStatusOverlay horizontal padding; "Saved to Photos" capsule horizontal padding |
| lg | 16 | TorchToggleButton `.padding(16)` (existing, unchanged) |
| xl | 20 | Torch button leading edge inset from screen edge |
| 2xl | 24 | Record button bottom inset above home indicator (existing, unchanged) |
| 3xl | 28 | Quality button and torch button bottom inset above home indicator |

Exceptions:
- Touch target minimum: 44pt for all tappable controls (QualitySettingsButton uses `frame(width:44, height:44)` — existing pattern; apply same to TorchToggleButton if it does not already meet 44pt minimum).
- Zoom preset buttons: `.padding(.horizontal, 10)` and `.padding(.vertical, 5)` — these non-scale values come from matching the iPhone Camera app capsule button proportions. Acceptable because they are sub-token presentational values, not layout values.

**Source:** Derived from existing codebase (`CameraContentView.swift`, `RecordingStatusOverlay.swift`, button files) + CONTEXT.md D-03 discretion for torch position + RESEARCH.md Pattern 3.

## Typography

All text in this phase uses `.system()` with named text styles or explicit sizes. Weights are limited to two choices project-wide: regular (400) and bold (700).

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Zoom preset label (inactive) | `.footnote` (~13pt) | regular (400) | n/a (single line) | Inactive zoom preset buttons: `1x`, `2x`, `3x` |
| Zoom preset label (active) | `.footnote` (~13pt) | bold (700) | n/a (single line) | Active zoom preset — same size, bold weight is the active-state signal |
| Recording timer | `.body` (~17pt) | bold (700) | n/a (single line) | `RecordingStatusOverlay` elapsed time — existing, unchanged |
| "Saved to Photos" banner | `.caption` (~12pt) | regular (400) | n/a (single line) | Success capsule — existing, unchanged |
| Control icons | `.title2` / `.title3` | — | n/a | SF Symbol icons in TorchToggleButton (`.title2`) and QualitySettingsButton (`.title3`) — existing, unchanged. Symbol rendering weight is controlled by SF Symbols configuration, not by the text typography contract. |

Active-state rule: the ONLY visual differentiator between active and inactive zoom preset buttons is font weight (bold vs regular) and foreground color (`.primary` vs `.white`). No separate background, no tint layer — this ensures the active signal is visible on both iOS 26 glassEffect and iOS 18–25 ultraThinMaterial. See RESEARCH.md Pitfall 5.

**Source:** Existing codebase + CONTEXT.md D-06 (match iPhone Camera app aesthetic) + RESEARCH.md Pattern 2 + Pitfall 5.

## Color

This is a camera overlay UI displayed on top of a live camera feed. "Dominant surface" is the live camera feed itself. Controls float above it with glass/material treatments.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | transparent / camera feed | Full-bleed background — `Color.black` base with live AVFoundation preview layers on top |
| Secondary (30%) | `.ultraThinMaterial` / `.glassEffect(.regular)` | Glass capsule/circle backgrounds on all camera controls |
| Accent (10%) | `Color.red` | Record button idle fill, blinking dot in RecordingStatusOverlay |
| Active-state | `Color.primary` | Active zoom preset text foreground (replaces `.white` on active button only) |
| Inactive controls | `Color.white` | All icon foregrounds, inactive zoom preset text |
| Destructive | `Color.red` | Same as accent — used only for the recording stop state indicator (not user-initiated destructive actions) |

Accent (`Color.red`) reserved for:
1. RecordButton inner fill circle (idle/ready-to-record state)
2. Blinking dot inside RecordingStatusOverlay (during active recording)

No other element uses `Color.red`.

Removed permanently: `Color.black.opacity(0.4)` (TorchToggleButton, QualitySettingsButton, ZoomLabelView) and `.black.opacity(0.6)` ("Saved to Photos" capsule). All replaced by glass modifier.

Glass API strategy (locked in STATE.md, D-09):
- iOS 26+: `.glassEffect(.regular, in: Shape())`
- iOS 18–25: `.background(.ultraThinMaterial, in: Shape())`
- Implementation: single shared `cameraGlassBackground(in:)` ViewModifier extension in `DualVideo/Shared/GlassBackground.swift`

**Source:** CONTEXT.md D-08, D-09, D-11 + RESEARCH.md Pattern 1 + STATE.md locked decisions.

## Layout Contract

### Bottom-Center Area (above home indicator)

```
[  ZoomPresetView  ]    ← HStack(spacing: 8): three glass capsule buttons
                           VStack spacing: 12pt between zoom row and RecordButton
[   RecordButton   ]    ← 72pt diameter, existing
     .padding(.bottom, safeAreaInsets.bottom + 24)
```

ZoomPresetView sits directly above RecordButton using a `VStack(spacing: 12)` in the bottom-center stack. The "Saved to Photos" success capsule appears between ZoomPresetView and RecordButton when visible.

### Bottom-Leading (Torch)

Torch toggle moves to `bottom-leading` — symmetric counterpart to the quality button.

```
VStack { Spacer(); HStack { TorchToggleButton; Spacer() } }
  .padding(.leading, 20)
  .padding(.bottom, safeAreaInsets.bottom + 28)
```

**Source:** CONTEXT.md D-03 (torch position is Claude's discretion) + RESEARCH.md Pattern 3. Rationale: bottom-leading mirrors the quality button at bottom-trailing, maintaining visual balance. Both controls are at thumb-reachable height.

### Bottom-Trailing (Quality)

```
VStack { Spacer(); HStack { Spacer(); QualitySettingsButton } }
  .padding(.trailing, 24)
  .padding(.bottom, safeAreaInsets.bottom + 28)
```

**Source:** CONTEXT.md D-02 + RESEARCH.md Pattern 3.

### Top-Center (Recording Status)

No change. `RecordingStatusOverlay` remains at:
```

```

## Interaction Contract

### Zoom Preset Buttons

| Interaction | Behavior |
|-------------|----------|
| Tap `1x` / `2x` / `3x` | Calls `cameraManager.setZoom(factor)` AND updates `activeZoomBase` to the selected factor |
| Pinch-to-zoom | Continues to work unchanged; updates active highlight when within ±0.25x of a preset |
| Active threshold | `abs(currentZoom - preset) < 0.25` — Claude's discretion (CONTEXT.md D-07) |
| Active visual | Bold weight + `.primary` foreground on matching preset button |
| Inactive visual | Regular weight + `.white` foreground |
| Double-tap reset | Existing — resets zoom to 1x, `activeZoomBase` → 1.0; no new behavior needed |

**Critical:** Tapping a preset MUST update `activeZoomBase` to prevent pinch snap (RESEARCH.md Pitfall 1).

### Glass Backgrounds

| Control | Shape | Glass modifier |
|---------|-------|---------------|
| TorchToggleButton | `Circle()` | `.cameraGlassBackground(in: Circle())` |
| QualitySettingsButton | `Circle()` | `.cameraGlassBackground(in: Circle())` |
| ZoomPresetView (each button) | `Capsule()` | `.cameraGlassBackground(in: Capsule())` |
| "Saved to Photos" capsule | `Capsule()` | `.cameraGlassBackground(in: Capsule())` |
| RecordingStatusOverlay | `Capsule()` | Already `.background(.ultraThinMaterial, in: Capsule())` — equivalent; no change |
| QualitySettingsSheet | sheet-level | `.presentationBackground(.ultraThinMaterial)` at call site |

### iOS Version Branching

Applied exactly once, in `GlassBackground.swift`:

```swift
extension View {
    @ViewBuilder
    func cameraGlassBackground<S: Shape>(in shape: S) -> some View {
        if #available(iOS 26, *) {
            self.glassEffect(.regular, in: shape)
        } else {
            self.background(.ultraThinMaterial, in: shape)
        }
    }
}
```

On iOS 26+, `ZoomPresetView` wraps the button row in `GlassEffectContainer` for merged-pill rendering:

```swift
if #available(iOS 26, *) {
    GlassEffectContainer(spacing: 8) { zoomButtonRow }
} else {
    zoomButtonRow
}
```

**Anti-pattern:** Do not stack `.glassEffect()` inside another glass view. Apply at the outermost control container only.

**Source:** RESEARCH.md Pattern 1 + Pitfall 2 + Pitfall 3.

## Anti-Patterns Banned in This Phase

The following patterns are explicitly prohibited and must not appear in any file after Phase 5:

| Pattern | Reason |
|---------|--------|
| `Color.black.opacity(0.4)` as a control background | Replaced by `cameraGlassBackground(in:)` |
| `.black.opacity(0.6)` as a control background | Same — includes "Saved to Photos" capsule |
| `ZoomLabelView` usage | Component deleted; use `ZoomPresetView` |
| Left-column `VStack` grouping torch + quality + zoom | Layout removed; controls repositioned individually |
| Applying `.glassEffect()` inside a label child view | Causes invisible tap areas — apply at Button level |
| Using `.glassProminent` with `.circle` shape on iOS 26 | Known rendering artifact — use `.regular` + foreground change |

**Source:** RESEARCH.md Anti-Patterns to Avoid.
