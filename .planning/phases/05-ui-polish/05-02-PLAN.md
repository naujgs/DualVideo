**TorchToggleButton.swift:**

Replace the two-modifier chain `.background(Color.black.opacity(0.4)).clipShape(Circle())` with a single `.cameraGlassBackground(in: Circle())` modifier.

The modifier must be applied at the `Button` level (after the label closure), not inside the label:

```swift
import SwiftUI

struct TorchToggleButton: View {
    let isTorchOn: Bool
    let onTap: () -> Void

    var body: some View {
        Button(action: onTap) {
            Image(systemName: isTorchOn ? "flashlight.on.fill" : "flashlight.off.fill")
                .font(.system(.title2, design: .default, weight: .medium))
                .foregroundStyle(isTorchOn ? Color.yellow : Color.white)
                .padding(16)
        }
        .cameraGlassBackground(in: Circle())   // applied at Button level, outside the label
        .accessibilityLabel(isTorchOn ? "Turn off torch" : "Turn on torch")
    }
}
```

Note: `.glassEffect()` on iOS 26 must be applied outside the Button label for correct tap area detection (RESEARCH.md Pitfall 3). The `.padding(16)` stays inside the label to size the icon correctly; the glass shape sizes from the Button's rendered frame.

**QualitySettingsButton.swift:**

Replace `.background(Color.black.opacity(0.4)).clipShape(Circle())` with `.cameraGlassBackground(in: Circle())`.

Apply at the Button level (after the label closure), not inside the `frame(width:44, height:44)`:

```swift
import SwiftUI

struct QualitySettingsButton: View {
    let isRecording: Bool
    let onTap: () -> Void

    var body: some View {
        Button(action: {
            guard !isRecording else { return }
            onTap()
        }) {
            Image(systemName: "slider.horizontal.3")
                .font(.system(.title3, design: .default, weight: .semibold))
                .foregroundStyle(.white)
                .frame(width: 44, height: 44)

        .cameraGlassBackground(in: Circle())   // applied at Button level
        .disabled(isRecording)
        .opacity(isRecording ? 0.5 : 1.0)
        .accessibilityLabel(isRecording ? "Video quality settings, unavailable" : "Video quality settings")
    }
}
```

Do NOT use `.glassProminent` — it causes known rendering artifacts with Circle() shape on iOS 26 (RESEARCH.md Anti-Patterns).

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-03 | Tampering | activeZoomBase sync in onPresetSelected | mitigate | Explicitly set `activeZoomBase = factor` in the onPresetSelected closure (RESEARCH.md Pitfall 1). Without this, a malformed zoom state allows the next pinch to produce zoom values outside the user's intent. The fix is already in the task action. |
| T-05-04 | Denial of Service | .presentationBackground(.ultraThinMaterial) on iOS 16.3 or lower | accept | Deployment target is iOS 18.0 — all supported devices have iOS 18+. `.presentationBackground` is iOS 16.4+. No risk within deployment floor. |
| T-05-05 | Spoofing | glassEffect modifier applied at wrong level | mitigate | Apply `.cameraGlassBackground(in:)` at the Button level (outside the label closure). Applying inside the label on iOS 26 causes invisible tap areas, allowing a confused deputy where the visual button is tappable in a different region than the glass shape. Mitigation: acceptance criteria verify modifier position. |
</threat_model>
