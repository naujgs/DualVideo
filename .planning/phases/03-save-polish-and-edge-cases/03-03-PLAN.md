func toggleTorch() {
    sessionQueue.async { [weak self] in
        guard let device = self?.backDevice,
              device.hasTorch,
              device.isTorchModeSupported(.on) else { return }
        do {
            try device.lockForConfiguration()

func syncSessionRunningState() {
    sessionQueue.async { [weak self] in

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CameraManager → AVCaptureDevice torch | Device hardware state; requires lockForConfiguration guard |
| RecordingManager → NotificationCenter | AVCaptureSession system notifications; background-queue origin |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-03-01 | Availability | Torch stays on after session interruption | mitigate | `turnTorchOff()` called in `handleInterruption()` before `stopRecording()`; battery drain prevented |
| T-03-03-02 | Tampering | `isTorchModeSupported(.on)` not checked | mitigate | Guard `device.hasTorch && device.isTorchModeSupported(.on)` in both `toggleTorch()` and `turnTorchOff()` (RESEARCH.md Pitfall 6) |
| T-03-03-03 | Denial of Service | interruptionEndedNotification handler crashes on nil cameraManager | mitigate | `[weak cameraManager]` capture list; `cameraManager?.syncSessionRunningState()` is no-op if nil |
| T-03-03-04 | Information Disclosure | Torch state not reflected in UI after OOB hardware change | accept | Hardware torch can be turned off by iOS system (overheating) without calling our handler; `isTorchOn` may become stale. Accepted: low-value cosmetic issue, no security or data impact |
</threat_model>
