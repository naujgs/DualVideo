## Spacing Scale

Declared values — multiples of 4 only, matching existing app conventions extracted from `CameraContentView.swift`, `TorchToggleButton.swift`, and `RecordingStatusOverlay.swift`:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4pt | Inline gaps (e.g. `HStack(spacing: 6)` in status overlay — nearest standard step) |
| sm | 8pt | Label horizontal padding (zoom label `.padding(.horizontal, 8)`) |
| md | 16pt | Button inner padding (torch button `.padding(16)`) |
| lg | 24pt | Bottom safe-area offset (`geo.safeAreaInsets.bottom + 24`) |
| xl | 32pt | Not yet used — available for settings sheet internal section gaps |
| 2xl | 48pt | Not yet used — available for sheet top padding |
| 3xl | 64pt | Not yet used — reserved for future layout gaps |

Exceptions:
- Touch target minimum: 44pt × 44pt (Apple HIG requirement). The torch button achieves this via 16pt padding around a `.title2` icon. Quality settings gear button must match the same 44pt minimum.
- RecordButton outer ring: 72pt — retained as-is, not changed in this phase.
- PiP corner radius: 12pt — retained.
- Control group vertical spacing: 8pt (`VStack(spacing: 8)` in torch + zoom column) — retain for quality indicator badge if added to that column.

## Typography

All fonts are SwiftUI system fonts via `.font(.system(...))`. No custom fonts are used in the app. (Source: all existing UI files.)

| Role | SwiftUI Spec | Approx Size | Weight | Line Height |
|------|-------------|-------------|--------|-------------|
| Body | `.system(.body)` | 17pt | regular (400) | 1.5 (system default) |
| Label / Caption | `.system(.caption)` | 12pt | regular (400) | 1.4 |
| Monospaced timer | `.system(.body, design: .monospaced).bold()` | 17pt | bold (700) | 1.5 |
| Footnote / HUD badge | `.system(.footnote, design: .monospaced, weight: .semibold)` | 13pt | semibold (600) | 1.3 |
| Control icon | `.system(.title2, design: .default, weight: .medium)` | 22pt | medium (500) | n/a (icon only) |

**Phase 4 additions:**
- Settings sheet section header: `.system(.subheadline, weight: .semibold)` — 15pt semibold. Matches iOS native sheet header conventions.
- Picker segment label: `.system(.body)` — 17pt regular. SwiftUI `Picker(.pickerStyle(.segmented))` renders its own label font; do not override.
- Trim time label (in/out point display): `.system(.footnote, design: .monospaced)` — 13pt regular. Consistent with ZoomLabelView pattern.
- File size hint (below bitrate picker): `.system(.caption)` — 12pt, `.secondary` foreground color. Informational only.

Maximum weights used in the app: regular (400) + bold/semibold for emphasis. Phase 4 adds no new weights — subheadline at semibold is already within the semibold tier used by ZoomLabelView.

## Color

This is a camera-first, full-bleed black UI. All controls float over the live camera preview. (Source: `CameraContentView.swift` — `Color.black` background, all controls use translucent overlays.)

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `Color.black` | Full-bleed camera background; sheet background inherits system `.background` |
| Secondary (30%) | `Color.black.opacity(0.4)` | Control pill/button backgrounds (torch, zoom label, save banner); settings sheet uses system `.systemGroupedBackground` (iOS dark/light adaptive) |
| Accent (10%) | `Color.red` | Active recording indicator dot, record button fill — reserved only for recording state |
| Accent alt | `Color.yellow` | Torch-on state only — reserved only for torch active state |
| Destructive | `Color.red` (same as recording accent) | No destructive actions in Phase 4 — not applicable |

**Accent reserved for:**
1. The blinking red dot in `RecordingStatusOverlay` — recording active state only.
2. The inner fill of `RecordButton` when idle (tap-to-record affordance).
3. No other use of `Color.red` is permitted in Phase 4 UI.
4. `Color.yellow` is reserved exclusively for `TorchToggleButton.isTorchOn == true`.

**Phase 4 new elements:**
- Quality settings sheet: uses system `SheetBackground` (`.ultraThinMaterial` or `.systemGroupedBackground`). Do not use `Color.black` for the sheet — it floats over the camera and must use a material/adaptive background so text remains legible in both light/dark environments.
- Quality indicator badge (if added to control column): matches `ZoomLabelView` pattern — `.black.opacity(0.4)` background, `.white` foreground, capsule shape.
- Trim sheet scrubber track: `Color.white.opacity(0.3)` for the inactive track, `Color.white` for the selected range. No accent color in the trimmer — the recording red would be semantically misleading here.
- Trim playhead / selected range highlight: `Color.white` — neutral, not red (red is reserved for recording state only).
- "Save Trimmed" button: `.white` label on `.black.opacity(0.4)` pill — consistent with existing control style. No accent color.
- "Save Full" secondary action: `.secondary` foreground (system gray) to visually de-emphasize compared to "Save Trimmed".

## Component Inventory

New components introduced in Phase 4, with visual contract:

### QualitySettingsButton

Entry point on the main camera UI to open quality settings.

- Shape: `Circle()`, 44pt × 44pt minimum touch target
- Icon: SF Symbol `gear` or `slider.horizontal.3`, `.system(.title3, weight: .medium)`
- Background: `Color.black.opacity(0.4)`, same as `TorchToggleButton`
- Foreground: `Color.white`
- Placement: Left control column, above or below `TorchToggleButton`, within `VStack(spacing: 8)`
- Disabled during recording: `true` — resolution cannot change mid-recording (per RESEARCH.md A1). Opacity 0.5 when disabled (matches `RecordButton` disabled state).
- Accessibility label: `"Video quality settings"`

### QualitySettingsSheet

Bottom sheet for resolution and bitrate selection.

- Presentation: `.sheet(isPresented:)` with `.presentationDetents([.height(240)])` and `.presentationDragIndicator(.visible)`
- Layout: `VStack(spacing: 24)` containing:
  1. Sheet handle (system drag indicator)
  2. Section header: "Resolution" — `.system(.subheadline, weight: .semibold)`, `.secondary` foreground
  3. `Picker("Resolution", selection:)` with `.pickerStyle(.segmented)` — options: "720p", "1080p"
  4. Section header: "Quality" — same style as Resolution header
  5. `Picker("Quality", selection:)` with `.pickerStyle(.segmented)` — options: "Low", "Medium", "High"
  6. File size hint: `Text("~{N} MB/min")` — `.system(.caption)`, `.secondary` foreground
- Internal padding: `.padding(.horizontal, 24)`, `.padding(.vertical, 16)`
- No explicit close button — sheet dismisses via drag or tap-outside. This matches the existing app's alert dismissal pattern.
- Cannot open during active recording — `QualitySettingsButton` is disabled when `recordingManager.phase == .recording`.

### TrimSheet

Full-height sheet displayed after `stopRecording()` when `pendingTrimURL` is non-nil.

- Presentation: `.sheet(item: $recordingManager.pendingTrimURL)` with `.presentationDetents([.large])` — full height required to show video preview.
- Layout (top to bottom):
  1. `VideoPlayer(player: avPlayer)` — fills top 60% of sheet. `AVKit.VideoPlayer` from SwiftUI. Player pauses automatically when sheet is dismissed.
  2. Trim range bar — custom view using two overlapping `Slider` controls. See TrimRangeBar below.
  3. In/out time labels — `HStack` with `Text(inTime)` `.leading` and `Text(outTime)` `.trailing`. Both use `.system(.footnote, design: .monospaced)`.
  4. Action buttons — `HStack(spacing: 16)`: "Save Trimmed" (primary) and "Save Full" (secondary).
- Background: system `.background` (adaptive) — trim sheet is a standard modal, not a camera overlay.
- No drag indicator — full-height sheet with explicit action buttons. User must choose an action to dismiss.

### TrimRangeBar

Two-thumb range slider for in/out point selection.

- Track height: 4pt
- Track background: `Color.white.opacity(0.3)` capsule
- Selected range highlight: `Color.white` capsule overlay
- Thumb: Circle, 24pt diameter, `Color.white` fill, `.shadow(color: .black.opacity(0.4), radius: 4)` — matches PiP shadow spec
- Implementation: Two `Slider(value:in:)` views with z-ordering and clamped `onChange` logic. In-slider capped at `outValue - minimumTrimDuration`. Out-slider floored at `inValue + minimumTrimDuration`. Minimum trim duration: 1.0 second.
- Accessibility: Each slider gets an `accessibilityLabel` — "Trim start" and "Trim end". Values announced as formatted time string (e.g. "0:03").

## Interaction Contract

### Quality Settings Entry Flow

1. User taps `QualitySettingsButton` (left control column, above torch) → `QualitySettingsSheet` slides up.
2. User changes resolution or bitrate picker → selection updates `VideoQualitySettings` in memory immediately.
3. Sheet dismisses (drag down) → `VideoQualitySettings.save()` called to persist to UserDefaults.
4. No confirmation required — settings are non-destructive preferences.

**Constraint:** `QualitySettingsButton` is `disabled(recordingManager.phase == .recording)`. Tapping has no effect and opacity is 0.5. No error shown — the disabled state is self-explanatory in context.

### Post-Recording Trim Flow

1. User taps Stop → `RecordingManager.stopRecording()` → finalization completes → `pendingTrimURL` set (non-nil).
2. `CameraContentView` detects `pendingTrimURL != nil` → `TrimSheet` appears.
3. User adjusts in/out sliders. `VideoPlayer` plays the clip for preview. Time labels update in real time.
4. "Save Trimmed" tap → `VideoTrimManager.trim(url:range:)` async → progress indicator (see below) → on success, `PhotoSaveManager.save()` → "Saved to Photos" banner (existing behavior).
5. "Save Full" tap → `PhotoSaveManager.save(originalURL)` directly → "Saved to Photos" banner.
6. Either action sets `pendingTrimURL = nil` after completion, dismissing the sheet.

**Trim export progress:** During `VideoTrimManager.trim()`, display an indeterminate `ProgressView()` centered over the `VideoPlayer` area. Disable both action buttons. Copy: "Trimming…" below the spinner, `.system(.caption)`, `.secondary`. No cancel button in Phase 4 — export is fast (passthrough, no re-encode).

**Error during trim export:** If `VideoTrimManager` throws, dismiss trim progress state and show the existing `"Save Failed"` alert pattern (same as `PhotoSaveManager` failure in Phase 3) with message: "Could not trim recording. Try saving the full clip instead." Actions: "Save Full" and "Dismiss".

### Disabled State During Recording

The following are disabled while `recordingManager.phase == .recording`:
- `QualitySettingsButton` — opacity 0.5, `disabled(true)`. No tooltip or explanation shown (screen is already dense).

Resolution and bitrate changes do not require stopping a recording in progress — the user simply cannot change them mid-recording. This matches the camera-operator mental model: configure before rolling, not during.

## State Machine — TrimSheet

```
pendingTrimURL = nil          → TrimSheet hidden
pendingTrimURL = url          → TrimSheet visible, idle
  User taps "Save Trimmed"   → TrimSheet .trimming (spinner, buttons disabled)
    export success             → pendingTrimURL = nil, "Saved to Photos" banner
    export failure             → TrimSheet .error, alert shown
      alert "Save Full"       → pendingTrimURL = nil, save full, "Saved to Photos" banner
      alert "Dismiss"         → TrimSheet idle (user can retry or tap "Save Full")
  User taps "Save Full"      → pendingTrimURL = nil, save full, "Saved to Photos" banner
```

---

## State Machine — QualitySettingsSheet

```
showQualitySettings = false    → sheet hidden
QualitySettingsButton tap      → showQualitySettings = true (only if phase != .recording)
  User adjusts pickers         → VideoQualitySettings updated in memory
  Sheet dismissed              → VideoQualitySettings.save() called
showQualitySettings = false    → sheet hidden, settings persisted
```
